# 🚀 System Design – FAANG Interview Resources

A curated, high-signal list of resources to master System Design for FAANG-level interviews.

---

# 📘 Foundational Learning

## 1️⃣ System Design Primer (Must Read)
https://github.com/donnemartin/system-design-primer

Covers scalability, load balancing, caching, databases, CAP theorem, and distributed systems fundamentals.

---

## 2️⃣ Designing Data-Intensive Applications (Book)
https://dataintensive.net/

Deep understanding of distributed systems, replication, consistency, and storage systems.

---

## 3️⃣ Grokking the System Design Interview
https://www.educative.io/courses/grokking-the-system-design-interview

Structured interview-style learning with real design problems.

# 🧠 Core Topics to Master

- Load Balancers
- Caching (Redis concepts)
- Database Indexing
- Replication & Sharding
- CAP Theorem
- Consistency Models
- Message Queues
- Rate Limiting
- Microservices vs Monolith
- API Design
- CDN & Edge Computing

# 🛠 Practice Design Problems

You should be able to design:

- URL Shortener
- Twitter Feed
- Chat Application
- Notification System
- Rate Limiter
- Video Streaming Service
- Food Delivery System

---

# 🎯 Goal

After completing these resources, you should be able to:

- Design scalable systems from scratch
- Explain tradeoffs clearly
- Justify architectural decisions
- Handle ambiguity confidently

System Design is not about memorizing diagrams.
It’s about structured thinking under constraints.
